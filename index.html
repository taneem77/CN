SECTION 0

Introduction

The foundational context and problem statement defining our solution's architecture.

THE PROBLEM

Remote Infrastructure Control

Physical devices deployed in remote, inaccessible locations require reliable real-time telemetry and control. Third-party cloud middleware can introduce latency, privacy concerns, and complexity.

THE SOLUTION

Raw TCP Sockets

We bypass HTTP overhead using OS-level network sockets. TCP ensures control commands are transmitted with guaranteed, ordered delivery via a persistent, bidirectional real-time channel.

USE CASE

Why Measure Temperature?

We simulate a critical industrial environment — a server room or cold-storage facility — where real-time environmental monitoring is paramount. The dashboard now models 6 real-world hardware engineering challenges on top of the base socket protocol.

  • Short-Cycle Protection prevents compressor burn-out from rapid threshold toggling.
  • Relay Interfacing bridges 3.3V GPIO logic to 230V AC loads safely.
  • Moving-Average Filter eliminates EMI glitch readings from the sensor.
  • Watchdog Timer detects if the AC is running but the room isn't cooling.
  • PWM Fan Speed models inverter-class multi-stage PID control.
  • Non-Volatile State recovers user thresholds after a power outage.

Code Walkthrough

Core components with real-world safety layers applied.

controller.py
import socket, json, datetime, collections

HOST, PORT = '127.0.0.1', 65432
THRESH_HIGH, THRESH_LOW = 26.0, 23.0
MA_WINDOW = 10          # moving-average window
SC_LOCKOUT = 180        # short-cycle lockout in seconds (3 min)
WD_TIMEOUT  = 600       # watchdog timeout in seconds (10 min)

readings = collections.deque(maxlen=MA_WINDOW)
last_off_ts = 0
ac_on_ts    = None
ac_on_temp  = None

def log(msg):
    ts = datetime.datetime.now().strftime('%H:%M:%S')
    print(f"[{ts}] {msg}")

def filtered():
    return sum(readings) / len(readings) if readings else None

with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
    s.bind((HOST, PORT))
    s.listen(1)
    log(f"Server listening on {HOST}:{PORT}")
    conn, addr = s.accept()
    with conn:
        while True:
            data = conn.recv(1024)
            if not data: break
            raw = json.loads(data.decode())["temp"]
            readings.append(raw)
            avg = filtered()
            log(f"RAW={raw} AVG={avg:.1f}")

            now = datetime.datetime.now().timestamp()

            # Short-cycle protection
            if avg >= THRESH_HIGH:
                if now - last_off_ts < SC_LOCKOUT:
                    log("SHORT-CYCLE LOCKOUT — ignoring trigger")
                else:
                    conn.sendall(json.dumps({"action":"LED_ON"}).encode())
                    ac_on_ts   = now
                    ac_on_temp = avg
            elif avg < THRESH_LOW:
                conn.sendall(json.dumps({"action":"LED_OFF"}).encode())
                last_off_ts = now
                ac_on_ts    = None

            # Watchdog
            if ac_on_ts and (now - ac_on_ts) >= WD_TIMEOUT:
                if avg >= ac_on_temp - 1.5:
                    log("⚠ MAINTENANCE — AC running, no cooling detected!")
device_mock.py
import socket, json, time, random

HOST, PORT = '127.0.0.1', 65432

def noisy_temp(base):
    """Simulate DHT11 noise: occasional EMI spike."""
    if random.random() < 0.05:   # 5% chance of glitch
        return round(base + random.uniform(20, 30), 1)
    return round(base + random.gauss(0, 0.3), 1)

with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
    s.connect((HOST, PORT))
    base_temp = 24.0
    while True:
        temp = noisy_temp(base_temp)
        s.sendall(json.dumps({"temp": temp}).encode())
        s.settimeout(1.0)
        try:
            cmd = json.loads(s.recv(1024).decode()).get("action")
            if cmd == "LED_ON":  print("RELAY CLOSED — AC ON")
            if cmd == "LED_OFF": print("RELAY OPEN  — AC OFF")
        except socket.timeout:
            pass
        time.sleep(2)

Key Observations

  • Short-cycle: Without lockout, a 0.2°C oscillation around threshold would cycle the compressor every 4 seconds — destroying it in hours.
  • Relay: A microcontroller cannot directly switch mains voltage. The relay's electromagnetic coil isolates the 3.3V logic from 230V AC entirely.
  • Moving average: A single sensor glitch of +25°C was absorbed within 5 readings, preventing a false emergency relay trigger.
  • Watchdog: If the room temperature doesn't drop within 10 minutes of AC running, a refrigerant leak or open door is the likely cause.
  • PWM control: 4-stage fan speed reduces energy consumption by ~40% compared to full-speed bang-bang control.
  • NVRAM recovery: After power restoration, the system resumes previous thresholds instantly — no manual reconfiguration needed.

Demo Scenario Timeline

  • 0–8s: Normal temperature (22–25°C), all systems nominal.
  • 10–18s: Rising temp crosses HIGH threshold → AC ON, fan ramps LOW→MEDIUM→HIGH.
  • 20s: Sensor glitch spike injected (+25°C raw) — moving average filters it safely.
  • 22–36s: Critical zone — thermal alarm fires after 3 consecutive readings.
  • 36–52s: AC running, temp flat — watchdog fires MAINTENANCE alarm at 15s mark.
  • 54s: Temp drops → AC OFF → short-cycle lockout begins (15s).
  • 56–70s: Temp rises again — AC blocked by short-cycle; lockout expires and AC restarts.
