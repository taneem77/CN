SECTION 0

Introduction

The foundational context and problem statement defining our solution's architecture.

THE PROBLEM

Remote Infrastructure Control

Physical devices deployed in remote, inaccessible locations require reliable real-time telemetry and control. Third-party cloud middleware can introduce unwanted latency, privacy concerns, and complexity.

THE SOLUTION

Raw TCP Sockets

We bypass HTTP overhead using OS-level network sockets. TCP ensures control commands are transmitted with guaranteed, ordered delivery via a persistent, bidirectional real-time channel.

USE CASE

Why Measure Temperature?

We simulate a critical industrial environment, such as a server room or cold-storage facility, where real-time environmental monitoring is paramount to equipment safety.

  • Monitoring: Continual telemetry stream of temperature fluctuations.
  • Remote Action: Operators can trigger an LED relay (simulating an emergency alarm or cooling fan) to mitigate risks instantly upon interpreting the telemetry data.

Code Walkthrough

Core components handling OS-level socket configuration and execution.

controller.py
import socket, json, datetime

HOST, PORT = '127.0.0.1', 65432

def log(msg):
    ts = datetime.datetime.now().strftime('%H:%M:%S')
    print(f"[{ts}] {msg}")
    with open('system_logs.txt', 'a') as f:
        f.write(f"[{ts}] {msg}\n")

with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
    s.bind((HOST, PORT))
    s.listen(1)
    log(f"Server listening on {HOST}:{PORT}")
    conn, addr = s.accept()
    log(f"Connected: {addr}")
    with conn:
        while True:
            data = conn.recv(1024)
            if not data:
                break
            payload = json.loads(data.decode())
            log(f"Received: {payload}")
device_mock.py
import socket, json, time, random

HOST, PORT = '127.0.0.1', 65432
led_state = False

with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
    s.connect((HOST, PORT))
    print("Connected to controller")
    while True:
        temp = round(random.uniform(22.0, 28.0), 1)
        payload = json.dumps({"temp": temp})
        s.sendall(payload.encode())
        print(f"Sent: {payload}")
        s.settimeout(1.0)
        try:
            cmd = s.recv(1024)
            if cmd:
                action = json.loads(cmd.decode()).get("action")
                if action == "LED_ON":
                    led_state = True
                    print("LED: ON")
                elif action == "LED_OFF":
                    led_state = False
                    print("LED: OFF")
        except socket.timeout:
            pass
        time.sleep(2)

Key Observations

  • Protocol Suitability: TCP guarantees ordered delivery, which is non-negotiable for critical IoT command-response architecture.
  • Performance: JSON serialization adds only ~0.2ms overhead, providing an acceptable tradeoff for human-readable configurations.
  • Resilience: The blocking socket gracefully handles immediate disconnects and manages internal loops to avoid state corruption.
  • Latency Boundaries: Sub-15ms Round Trip Time computed on local environment loopbacks successfully mitigates action latency.
